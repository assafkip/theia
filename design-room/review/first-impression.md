# Floor — first-impression (50ms / 3-second scan)

- **cited (Lindgaard 2006):** first aesthetic judgment forms in ~50ms and anchors
  trust. The MVP's first screen is a dark form (key field + Choose PDF). It reads as
  a config panel, not a product with a point of view. Gap: in the first screen the
  eye must land on WHAT THIS DOES + the proof idea, not on an input.
- **cited (F/Z-pattern + hierarchy):** primary action ("drop a PDF") and the one-idea
  line must sit on the first-screen scan path. Constraint on the LEAD's subject: the
  hero figure must not push the drop-target below the fold — this is a tool, the
  BEFORE state has to invite the drop within the first screen.
- **grounded:** both references put the value line + primary action high; Kali's CTA
  and Maltego's H1 are the first fixations. The build must match that (value first,
  chrome second).
